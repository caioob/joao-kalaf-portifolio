## 3. Deviations from spec

1. **Test file is `i18n.test.jsx`**, not the `i18n.test.js` named in architecture §2 — it renders JSX for the provider integration tests, and Vite only transforms JSX in `.jsx` files.
2. **`App.jsx` now calls `getProfile()`** — the step 2 plan deferred all data-layer wiring to step 5, but the designer's name is content (not a UI string), so leaving it hardcoded would have contradicted FR-3's "no hardcoded copy". One read-only call; full wiring still lands in step 5.

## 4. Issues found & fixed

- **`localStorage` was `undefined` in tests** (7 failures): Node 26 ships an experimental global `localStorage` that is `undefined` without `--localstorage-file`, and it shadows jsdom's implementation when Vitest populates globals. Fixed three ways at once: source/tests always use `window.localStorage` explicitly; the `test`/`test:watch` scripts set `NODE_OPTIONS=--no-experimental-webstorage`; and the jsdom environment gets an explicit `url` (`vite.config.js`) since storage needs a non-opaque origin. Recorded in CLAUDE.md so it isn't rediscovered.

## 5. Decisions of record

- `translate()` accepts an injectable dictionaries argument (default: real ones) purely so the EN-fallback path is testable — key parity makes that path unreachable with the real dictionaries.
- `I18nProvider` lives inside `App` (not `main.jsx`) so `App` is self-contained and tests don't need wrapping.
- Missing translation keys render the key itself rather than crashing or rendering blank — visible in review, harmless in production.
